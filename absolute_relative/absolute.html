An absolute link is one that points to any location that uses the following struction for its destination:

scheme://domain/path

The scheme indicates the protocol that the browser must use to request the resource, for example http or https.

The :// is a character pattern that acts as a delimiter between the scheme and the domain. It has no special or inherent meaning. In fact, Tim Berners-Lee (the inventor of the World Wide Web, HTML, the URL system, and HTTP) decided to use this combination of symbols simply because the // was already familiar to him, and the : was more easily human-readable. However, he admits that one small change he would make - if he could do it over again - would have been to drop the // altogether and just use the : after http in URLs!

The path, or more specifically the path to the resource, is kind of a map that points us directly to the content we would like to see. On a static website, this is literally the file structure of the folders and documents stored on the server. On a dynamic website, this is a logical path that is interpreted by the software that is delivering the content.

There may be a huge string of apparent gibberish that is appended to some URLs. These parameters are often key/value pairs separated with the & symbol. Web servers will use this information to perform some voodoo behind the scenes before presenting the resource to you. You may also see an anchor to a specific section of the resource, identified in the URL by the #.
